Branch

En git los branch o ramas son entornos o espacios independientes para trabajar sobre el mismo sin modificar el conjunto de archivos originales

A la hora de desarrollar, cada uno crea su rama. El tronco principal es el main, y las ramas el branch.

Es importante crear ramas hijas al main, o master. Cuando se fusionan ramas al main, se tiene que tener la precaución de que se integren
